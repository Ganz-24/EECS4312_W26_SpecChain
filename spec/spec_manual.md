# Requirement ID: FR1

- Description: [The system shall provide access to the core meditation library without requiring a paid subscription.]
- Source Persona: [P1]
- Traceability: [Derived from review group G1]
- Acceptance Criteria: [Given a user who has not paid for a subscription When the user opens the core meditation library Then the user can play core sessions without a payment gate.]

# Requirement ID: NFR1

- Description: [The system shall present donation requests as optional actions that do not interrupt active meditation playback.]
- Source Persona: [P1]
- Traceability: [Derived from review group G1]
- Acceptance Criteria: [Given a user who is in an active meditation session When a donation option is shown Then the session continues without any blocking pop-up or forced purchase step.]

# Requirement ID: FR2

- Description: [The system shall provide a clearly labeled beginner section that contains guided introductory sessions.]
- Source Persona: [P2]
- Traceability: [Derived from review group G2]
- Acceptance Criteria: [Given a first-time user on the home screen When the user looks for beginner content Then a beginner section is visible and contains at least one guided introductory session.]

# Requirement ID: NFR2

- Description: [The system shall allow a new user to reach a beginner meditation session within three taps from the home screen.]
- Source Persona: [P2]
- Traceability: [Derived from review group G2]
- Acceptance Criteria: [Given a first-time user on the home screen When the user navigates to a beginner session Then the user can start a beginner session within three taps.]

# Requirement ID: FR3

- Description: [The system shall provide meditation content categorized for sleep, stress, and anxiety support.]
- Source Persona: [P3]
- Traceability: [Derived from review group G3]
- Acceptance Criteria: [Given a user browsing the library When the user opens the category listings Then categories for sleep, stress, and anxiety are available.]

# Requirement ID: NFR3

- Description: [The system shall continue audio playback for a meditation session when the screen turns off or audio accessories are connected.]
- Source Persona: [P3]
- Traceability: [Derived from review group G3]
- Acceptance Criteria: [Given a session that is already playing When the device screen turns off or earbuds are connected Then playback continues without restarting or freezing.]

# Requirement ID: FR4

- Description: [The system shall track completed meditation sessions and update the user's streak or consistency record after a completed session.]
- Source Persona: [P4]
- Traceability: [Derived from review group G4]
- Acceptance Criteria: [Given a user who completes a meditation session When the session ends successfully Then the streak or consistency record updates to reflect the completion.]

# Requirement ID: FR5

- Description: [The system shall allow a user to view their recent meditation progress, including completed sessions and current streak or consistency status.]
- Source Persona: [P4]
- Traceability: [Derived from review group G4]
- Acceptance Criteria: [Given a logged-in user When the user opens the progress area Then the app shows completed sessions and the current streak or consistency status.]

# Requirement ID: FR6

- Description: [The system shall synchronize user progress and streak data across signed-in devices.]
- Source Persona: [P5]
- Traceability: [Derived from review group G5]
- Acceptance Criteria: [Given a signed-in user on two devices When the user completes a session on the first device and refreshes the second device Then the progress and streak data match on both devices.]

# Requirement ID: FR7

- Description: [The system shall provide offline playback for meditation content that has been downloaded by the user.]
- Source Persona: [P5]
- Traceability: [Derived from review group G5]
- Acceptance Criteria: [Given a user who downloaded a meditation session When the device has no network connection Then the downloaded session can still be opened and played.]
